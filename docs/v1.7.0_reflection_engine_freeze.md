# RationaleVault Architecture Freeze — v1.7.0 Reflection Engine

This document locks the public API surface, rules, evaluation formulas, compiler transforms, and projection behaviors for RationaleVault's **Reflection Engine** (Epic F1). It enforces deterministic behavior in deriving and consolidating execution feedback without AI intervention.

---

## 1. Frozen Public API Surface

The following classes, interfaces, and rule sets are frozen under v1.7.0 and must not be altered:

### Engine Components
* **`ReflectionAssessment`**: Holds rule evaluation outputs, scoring parameters, duplicate suppression status, decision reasonings, source records, and explicit rule provenance (`passed_rules`, `failed_rules`, `triggered_rules`, and `rule_scores`).
* **`ReflectionCandidateBuilder`**: Ephemeral builder grouping learning records by target and deriving dominant triggers (`PERFORMANCE_DEGRADATION`, `LOW_CONFIDENCE`, `OUTCOME_MISMATCH`).
* **`ReflectionRule`**: Abstract base class defining evaluation policies.
* **`ReflectionRuleEngine`**: Orchestrates sequential evaluation of rule pipelines.
* **`ReflectionCompiler`**: Compiles assessment metadata and candidate properties into final immutable `Reflection` entities.

### Deterministic Rules
* **`MinSupportingRecordsRule`**: Enforces a minimum supporting records count before initiating reflection.
* **`RecurrenceThresholdRule`**: Verifies that the targeted issue has occurred a specific threshold of times.
* **`ConflictingEvidenceRule`**: Rejects candidates if the number of success (conflicting) learning records matches or exceeds failure (supporting) records.
* **`MinConfidenceRule`**: Determines whether the calculated confidence score satisfies the threshold configuration parameter.
  * **Confidence Formula**:
    $$confidence = \min(1.0, \max(0.0, base\_confidence + recurrence\_bonus - conflict\_penalty))$$
    Where:
    * $base\_confidence = 0.5$
    * $recurrence\_bonus = \min(0.4, (recurrence\_count - 1) \times 0.1)$
    * $conflict\_penalty = conflict\_count \times 0.15$
* **`DuplicateSuppressionRule`**: Rejects candidate evaluations if an active/completed reflection already exists for the target.

---

## 2. Frozen Projection

* **`ReflectionStateProjection`**: Derived projection mapped to build priority `95` producing consolidated `ReflectionReport` instances from historical ledger events and `ExecutionLearningRecord` collections.

---

## 3. Unfrozen Configurations (Allowed to Evolve)
* Epsilon bounds for conflict penalties or confidence bonuses.
* Minimum supporting record counts and recurrence threshold boundaries.
* Natural language formatting inside synthesized insights and actionable guidelines.
