# v3.0 Remote Sessions Freeze

**Date:** 2026-06-28  
**Version:** v3.0.0  
**ADR:** ADR-021

## Summary

Remote Sessions introduces the contracts for distributed cognition: multiple runtime nodes sharing workspaces, agents migrating between nodes, and telemetry aggregated across nodes. Scope is contracts-only — no networking, no actual distributed coordination.

## Frozen Contracts

### RuntimeNode (RSES-NODE-[hash])
- Identity of a runtime instance
- Records transport type, endpoint, capabilities
- Separate from NodeHealth (identity ≠ health)

### NodeHealth
- Operational health of a runtime node
- Heartbeat, latency, load, memory pressure, telemetry delay
- Evolves independently from RuntimeNode

### RemoteSession (RSES-RS-[hash])
- Agent session spanning multiple nodes
- Tracks current node, handoff count, status
- Migration history reconstructed from SessionHandoff records

### SessionHandoff (RSES-HO-[hash])
- Immutable record of session transfer
- State machine: INITIATED → IN_PROGRESS → COMPLETED | FAILED
- Retries produce new records, not state mutations

### NodeRegistry (RSES-NR-[hash])
- Registry of all known runtime nodes with health
- Immutable snapshot — mutations produce new instances
- Queries: online_nodes, get_nodes_for_workspace

### CrossNodeTelemetry (RSES-XTEL-[hash])
- Telemetry aggregated from multiple nodes
- Records source nodes, aggregation method, sample count

### AggregationStrategy Protocol
- Pluggable components for telemetry aggregation
- H7 ships with 5 strategies: Average, Sum, Min, Max, P95
- New strategies are plugins

### NodeTelemetryAggregator
- Aggregates telemetry from multiple nodes
- Accepts pluggable aggregation strategies

## Distributed Capabilities

Five new values in the Capability enum:
- CAN_HOST_REMOTE
- CAN_MIGRATE
- CAN_AGGREGATE_TELEMETRY
- CAN_SIMULATE
- CAN_VIEW_POLICY
