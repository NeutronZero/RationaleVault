# v2.9 Policy Simulator Freeze

**Date:** 2026-06-28  
**Version:** v2.9.0  
**ADR:** ADR-020

## Summary

The Policy Simulator adds deterministic replay-based simulation to the adaptive memory system. It replays historical telemetry through candidate policies and estimates the outcome, producing evidence for the Adaptive Policy Engine.

## Frozen Contracts

### MetricEstimate
- Single estimated metric value with confidence
- Extensible — new metrics are added as new keys without changing this contract

### SimulationResult (PSIM-RES-[hash])
- Estimated metrics for a single policy under replay
- Metrics stored as `dict[str, MetricEstimate]` for forward compatibility
- Includes sample count and overall confidence

### SimulationScenario (PSIM-SCN-[hash])
- Defines what is being simulated
- Identity (scenario_id) separate from presentation (name)
- Auto-generated names from policy names when not supplied

### SimulationReport (PSIM-[hash])
- Complete comparison of two policies
- Per-policy estimated metrics
- Dimension-level deltas
- Classification of improvements and degradations
- Human-readable explanation
- Overall delta score and confidence

### DimensionEstimator Protocol
- Pluggable components (one per policy dimension)
- H6.5 ships with three: RetrievalEstimator, CacheEstimator, ProvenanceEstimator
- New estimators (Write, Dedup, Cost) are plugins

### PolicyReplayEngine
- Stateless replay engine accepting `Iterable[PolicyTelemetry]`
- Orchestrates estimators across telemetry
- Returns SimulationResult

### PolicySimulator
- Produces SimulationReports from two policies
- Computes deltas, classifications, confidence, explanation
- Never knows about thresholds, rules, or cooldowns

## Identifier Prefix

| Prefix | Family | Purpose |
|--------|--------|---------|
| PSIM | Simulation | Reports, results, scenarios |

PSIM uses sub-prefixes:
- PSIM-[hash] — top-level reports
- PSIM-RES-[hash] — per-policy results
- PSIM-SCN-[hash] — scenarios

## Confidence Factors

1. **Sample factor**: `min(1.0, sample_count / 100)`
2. **Metric factor**: `len(metrics_present) / len(all_metrics)`
3. **Time factor**: temporal coverage of telemetry data
4. **Policy distance**: penalty for large policy divergences (0.0-0.3)

## Extension Points

New estimators can be added by implementing `DimensionEstimator`:
- `WriteEstimator` — write validation and importance thresholds
- `DedupEstimator` — deduplication strategy effectiveness
- `CostEstimator` — token cost and API usage
- `ConnectorEstimator` — connector-specific latency and reliability
