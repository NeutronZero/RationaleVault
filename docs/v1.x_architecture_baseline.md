This document serves as the release constitution for the RationaleVault v1.x series. It aggregates the established freezes, database naming identifiers, projection dependency graphs, and test baselines.

## 2. Standardized Identifiers (Relay Namespace)

Every dynamic record, belief, execution, or learning state is mapped via a deterministic, prefix-based identifier. Below is the canonical registry:

| Prefix | Domain | Generated By | Stable Format |
| :--- | :--- | :--- | :--- |
| `BEL-` | Belief Domain | Belief Builder | `BEL-[hash]` |
| `CONTR-` | Contradiction Domain | Contradiction Engine | `CONTR-[hash]` |
| `SYN-` | Synthesis Domain | Cognitive Head | `SYN-[hash]` |
| `DEC-` | Decision Domain | Cognitive Head | `DEC-[hash]` |
| `SKL-` | Skill Manifest Domain | Skill Manifest | `SKL-[hash]` |
| `SKE-` | Skill Execution | Skill Runtime | `SKE-[hash]` |
| `SRT-` | Skill Result | Skill Executor | `SRT-[hash]` |
| `ART-` | Promoted Artifact | Execution Gate Promoter | `ART-[hash]` |
| `LEARN-` | Execution Learning Record | Learning Record Builder | `LEARN-[hash]` |
| **`REFL-`** | *Reserved for future Reflection* | *Reflection Engine* | *REFL-[hash]* |

## 3. Projection Dependency Graphs

Determinism is guaranteed by resolving the event stream in a strict sequence. The current frozen projection DAG is laid out below:

### Core Cognitive Stream
```text
Event Ledger → MemoryProjection → KnowledgeProjection → GraphProjection
```

### Execution Telemetry Stream
```text
Event Ledger → ExecutionStateProjection → ExecutionAnalyticsProjection → ExecutionIntelligenceProjection
```
