# RationaleVault Vendor SDK Freeze — v2.4

**Effective**: 2026-06-26
**Scope**: Vendor SDK extension layer (H2)

---

## Architecture

```
Agent Runtime
    ↕ AgentSession
Transport SDK (v2.3)
    ↕ TransportSession
Vendor SDK (v2.4)
    ↕ VendorAdapter
Agent Implementations
    ↕ VendorAdapters
```

### Layer Separation

| Layer | Role | Frozen? |
|-------|------|---------|
| **Transport SDK** | HOW data moves | v2.3 (platform) |
| **Vendor SDK** | WHAT data means | v2.4 (extension) |
| **Agent Implementations** | Concrete integrations | Extension |

### Extension vs Platform

- **Platform** (frozen): Transport contracts, transport adapters, runtime.
- **Extension** (this doc): Vendor manifests, capability mapping, vendor adapters.

## Contracts

### VendorManifest (VMNF-[hash])

Immutable identity for a vendor:

```
VendorManifest {
    manifest_id: str            # VMNF-[hash]
    name: str                   # Human-readable
    vendor_id: str              # "anthropic", "openai", "google"
    version: str                # SemVer
    supported_transports: list[TransportType]
    supported_capabilities: frozenset[Capability]
    default_serializer: str
    status: VendorStatus        # ACTIVE | DEPRECATED | EXPERIMENTAL
    metadata: dict[str, str]
}
```

### CapabilityMapping

Deterministic bidirectional mapping:

```
Vendor → Runtime:  vendor_to_runtime["read"] = Capability.READ_WORKSPACE
Runtime → Vendor:  runtime_to_vendor(Capability.READ_WORKSPACE) = "read"
```

Reverse map is auto-built from `vendor_to_runtime` dict.

### VendorAdapter (ABC)

Every vendor adapter implements:

```python
class VendorAdapter(ABC):
    def manifest(self) -> VendorManifest: ...
    def capability_mapping(self) -> CapabilityMapping: ...
    def serialize(self, package_dict: dict) -> bytes: ...
    def content_type(self) -> str: ...
    def format_name(self) -> str: ...
    def supported_transports(self) -> list[TransportType]: ...
    def is_compatible(self, transport_type: TransportType) -> bool: ...
```

## What's Frozen

- `VendorManifest` schema and generation algorithm
- `CapabilityMapping` bidirectional mapping
- `VendorAdapter` ABC interface
- `VendorStatus` enum values

## What's NOT Frozen

- Concrete vendor adapters (vendor implementations)
- Vendor-specific serialization formats
- Vendor capability mappings (vendor-specific data)

## Migration Policy

- Concrete vendor adapters are vendor-specific, not platform-managed.
- New vendors are added as extensions, not core changes.
- Vendor deprecation follows `VendorStatus.DEPRECATED` lifecycle.
