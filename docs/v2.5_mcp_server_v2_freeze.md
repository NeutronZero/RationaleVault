# RationaleVault MCP Server v2 Freeze — v2.5

**Effective**: 2026-06-26
**Scope**: MCP Server v2 (H3)

---

## Architecture

MCP Server v2 routes through Workspace + Agent Runtime instead of raw projections.

```
MCP Client (IDE, CLI, Agent)
    ↕ MCP Protocol (stdio / SSE)
MCP Server v2 (v2.5)
    ↕ MCPToolRegistry + MCPRuntime
Workspace Layer (v2.1)
    ↕ WorkspaceContext + WorkspacePackage
Agent Runtime (v2.2)
    ↕ AgentSession + Capabilities
Transport SDK (v2.3)
    ↕ TransportSession
Vendor SDK (v2.4)
    ↕ VendorAdapter
Agent Implementations
```

### Key Changes from v1

| v1 MCP Server | v2 MCP Server |
|---------------|---------------|
| Calls raw projections directly | Routes through WorkspaceContext |
| No session management | AgentSession via AgentRuntime |
| Stateless tool calls | STATELESS / SESSION / STREAMING modes |
| Raw dict returns | MCPToolResult standardized |
| No capability checks | Capability-gated tools |
| Manual tool registration | MCPToolRegistry with bindings |

## Contracts

### MCPToolDef (MCPT-[hash])

Immutable tool definition:

```
MCPToolDef {
    tool_id: str                # MCPT-[hash]
    name: str
    description: str
    category: MCPToolCategory   # READ | WRITE | STREAM | ADMIN
    input_schema: dict
    output_schema: dict
    required_capabilities: frozenset[str]
    session_mode: MCPSessionMode  # STATELESS | SESSION | STREAMING
}
```

### MCPBinding (MCPB-[hash])

Binds a tool to a workspace and optional agent session:

```
MCPBinding {
    binding_id: str             # MCPB-[hash]
    tool_id: str
    workspace_id: str
    agent_session_id: str | None
    vendor_id: str | None
}
```

### MCPToolResult

Standardized result from every tool invocation:

```
MCPToolResult {
    tool_id: str
    success: bool
    data: dict
    error: str | None
    context_hash: str | None
    workspace_package_ref: str | None
}
```

### MCPManifest (MCPM-[hash])

Server identity and capability advertisement:

```
MCPManifest {
    manifest_id: str            # MCPM-[hash]
    server_name: str
    version: str
    tools: list[MCPToolDef]
    supported_transports: list[str]
}
```

### MCPToolRegistry

Manages tool definitions, handlers, and bindings:

```python
registry.register_tool(name, description, category, handler, ...)
registry.get_tool_by_name(name)
registry.list_tools(category)
registry.create_binding(tool_id, workspace_id, ...)
registry.get_bindings_for_workspace(workspace_id)
```

### MCPRuntime

Orchestrates tool execution:

```python
runtime.execute_tool(tool_name, arguments, workspace_context)
runtime.execute_tool_with_binding(binding_id, arguments, workspace_context)
runtime.build_manifest(server_name, version)
```

## What's Frozen

- MCPToolDef schema and generation algorithm
- MCPBinding schema and generation algorithm
- MCPToolResult contract
- MCPManifest schema and generation algorithm
- MCPToolRegistry interface
- MCPRuntime execution contract

## What's NOT Frozen

- Concrete tool implementations (tools are contributed)
- MCP protocol wire format (handled by `mcp` library)
- Transport-specific details (stdio vs SSE)
