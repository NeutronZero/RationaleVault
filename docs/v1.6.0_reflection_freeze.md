# RationaleVault Architecture Freeze — v1.6.0 Reflection Contracts

This document locks the public API surface, data models, serialization contracts, and identifier schemas for RationaleVault's **Reflection Subsystem** (Contracts F0). It establishes formal boundaries for triggers, candidates, reflection execution events, insight synthesis, and reporting.

---

## 1. Frozen Public API Surface

The following contracts and schemas are frozen under v1.6.0 and must not be altered:

### Enum Definitions
* **`ReflectionReason`**: Core categories triggering a reflection event. Supported values:
  * `OUTCOME_MISMATCH`
  * `CONTRADICTION_DETECTED`
  * `LOW_CONFIDENCE`
  * `GATE_BLOCK`
  * `MANUAL_TRIGGER`
  * `PERFORMANCE_DEGRADATION`
* **`ReflectionStatus`**: Lifecycle states of a reflection run. Supported values:
  * `PENDING`
  * `ACTIVE`
  * `COMPLETED`
  * `REJECTED`
  * `ABORTED`

### Dataclass Contracts
* **`ReflectionConfig`**: Configuration boundary containing policy version, enablement state, confidence thresholds, and active trigger reasons.
* **`ReflectionCandidate`**: Immutable snapshot representing a candidate event proposed for reflection.
  * **ID Format**: `RCAND-[hash]` where the hash is the first 8 characters of the SHA-256 hash of:
    `candidate_reflection:{source_artifact_id}:{reason}:{config_version}`
* **`Reflection`**: Durable event-sourced reflection object containing insights, reconstructed rationale, and actionable guidelines.
  * **ID Format**: `REFL-[hash]` where the hash is the first 8 characters of the SHA-256 hash of:
    `reflection:{candidate_id}:{created_at}`
* **`ReflectionReport`**: Aggregated audit report compile containing reflection instances and summary metrics.
  * **ID Format**: `RREP-[hash]` where the hash is the first 8 characters of the SHA-256 hash of:
    `report:{sorted_reflection_ids_comma_separated}:{created_at}`

---

## 2. Frozen Dataflow & Dependency DAG

The processing flow for the Reflection Subsystem must follow a strict, uni-directional pipeline. Downstream modules must not form cyclical dependencies or bypass intermediate states:

```text
Artifact / Event Store
          │
          ▼
ReflectionCandidate (RCAND-[hash])
          │
          ▼
    Reflection (REFL-[hash])
          │
          ▼
ReflectionReport (RREP-[hash])
```

---

## 3. Unfrozen Subsystem Configurations (Allowed to Evolve)

The following components sit on the policy/implementation side of the architecture boundary and are free to evolve without violating the v1.6.0 freeze:
* Default confidence threshold levels (e.g. tuning the float bounds).
* Prompts, templates, and instruction sets used to extract insights or reconstruct rationales.
* Logging formats and non-functional execution metadata.
* Poll rates or trigger-matching scheduler intervals.
