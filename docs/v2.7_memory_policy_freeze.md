## Architecture

The Memory Policy Engine sits behind the MemoryBroker, controlling all memory behavior.

```
MemoryBroker
    ↕
MemoryPolicyEngine
    ↕
┌─────────────────────────────────────┐
│         MemoryPolicy (composite)    │
│  ├── RetrievalPolicy (MPOL-RET-)    │
│  ├── CachePolicy (MPOL-CAC-)        │
│  ├── ProvenancePolicy (MPOL-PRV-)   │
│  ├── WritePolicy (MPOL-WRT-)        │
│  └── DedupPolicy (MPOL-DUP-)        │
└─────────────────────────────────────┘
    ↕
Memory Layer (BM25, Semantic, RRF)
```

### Preset Policies

| Policy | Use Case |
|--------|----------|
| `default` | Conservative, safe for all scenarios |
| `aggressive` | High-throughput, relaxed dedup and caching |
| `strict` | Audit-critical, full provenance and validation |

```

    strategy: RetrievalStrategy     # LEXICAL | SEMANTIC | HYBRID | GRAPH | ADAPTIVE

```


### WritePolicy (MPOL-WRT-[hash])

```
WritePolicy {
    validation: WriteValidation      # NONE | SCHEMA | IMPORTANCE | FULL
    min_importance: str
    max_content_length: int
    require_project_id: bool
    require_tags: bool
    allowed_memory_types: frozenset[str] | None

```

### DedupPolicy (MPOL-DUP-[hash])

```
DedupPolicy {
    strategy: DedupStrategy          # EXACT | SIMILARITY | SEMANTIC | NONE
    similarity_threshold: float
    merge_on_dedup: bool
    log_dedup_events: bool
}
```

### MemoryPolicy (MPOL-[hash])

Composite policy combining all five dimensions.

### MemoryPolicyEngine

```python
engine = MemoryPolicyEngine(policy=MemoryPolicy.default())
result = engine.validate_write(request)      # Write validation
is_dup, dup_id = engine.check_dedup(...)    # Deduplication check
enriched = engine.enforce_provenance(result) # Provenance enforcement
shaped = engine.shape_retrieval(results)     # Retrieval shaping
valid = engine.is_cache_valid(ts, now)       # Cache validation
```

## What's Frozen

- All five policy schemas and generation algorithms
- MemoryPolicy composite schema
- MemoryPolicyEngine interface (validate_write, check_dedup, enforce_provenance, shape_retrieval, should_cache, cache_key, is_cache_valid)
- Preset policies (default, aggressive, strict)

## What's NOT Frozen

- Memory retrieval internals
- Memory synthesis pipeline
- Cache storage implementation
- Policy selection logic (agent-controlled)
