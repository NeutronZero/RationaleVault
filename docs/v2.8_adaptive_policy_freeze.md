**Effective**: 2026-06-26
**Scope**: Adaptive Policy Engine (H6)

## Architecture

The Adaptive Policy Engine closes the loop between evaluation metrics and runtime behavior.

```
Evaluation Framework
      ↓
Telemetry (TELE-[hash])
      ↓
TelemetryCollector
      ↓
RuleEvaluator
      ↓
PolicyAdjustment (PADJ-[hash])
      ↓
AdaptivePolicyEngine
      ↓
Adapted MemoryPolicy
      ↓
MemoryBroker
      ↓
Better Evaluation Results
```

### Feedback Loop

```
Evaluation → Telemetry → Rule Evaluation → Policy Adjustment → Better Results
```

## Contracts

### PolicyTelemetry (TELE-[hash])

```
PolicyTelemetry {
    telemetry_id: str              # TELE-[hash]
    metric_type: MetricType
    value: float
    sample_count: int
    window_seconds: int
}
```

### MetricType

| Metric | Description |
|--------|-------------|
| RETRIEVAL_PRECISION | Accuracy of retrieval results |
| RETRIEVAL_LATENCY_MS | Retrieval time in milliseconds |
| CACHE_HIT_RATE | Fraction of cache hits |
| DEDUP_RATE | Fraction of duplicates detected |
| PROVENANCE_COVERAGE | Fraction of results with provenance |
| WRITE_SUCCESS_RATE | Fraction of successful writes |
| RESULT_COUNT_AVG | Average number of results returned |
| CONFIDENCE_SCORE_AVG | Average confidence score |

### AdjustmentRule (ARUL-[hash])

```
AdjustmentRule {
    rule_id: str                   # ARUL-[hash]
    metric_type: MetricType
    dimension: PolicyDimension
    direction: AdjustmentDirection  # INCREASE | DECREASE | MAINTAIN
    threshold: float
    adjustment_magnitude: float
    min_bound: float
    max_bound: float
    cooldown_seconds: int
}
```

### PolicyAdjustment (PADJ-[hash])

```
PolicyAdjustment {
    adjustment_id: str             # PADJ-[hash]
    rule_id: str
    dimension: PolicyDimension
    direction: AdjustmentDirection
    current_value: float
    recommended_value: float
    metric_value: float
    confidence: float
    reason: str
}
```

### AdaptivePolicy (APOL-[hash])

```
AdaptivePolicy {
    policy_id: str                 # APOL-[hash]
    name: str
    base_policy_name: str
    rules: list[AdjustmentRule]
    enabled: bool
    max_adjustments_per_window: int
    evaluation_window_seconds: int
}
```

### AdaptivePolicyEngine

```python
engine = AdaptivePolicyEngine(adaptive_policy=AdaptivePolicy.default())
engine.record_telemetry(telemetry)
adjustments = engine.evaluate(current_values)
adapted_policy = engine.apply_adjustments(base_policy, adjustments)
summary = engine.get_metric_summary()
```

### TelemetryCollector

Append-only telemetry storage with aggregation.

### RuleEvaluator

Evaluates rules against telemetry, respects cooldowns and bounds.

## What's Frozen

- PolicyTelemetry schema and generation algorithm
- MetricType enum values
- AdjustmentRule schema and generation algorithm
- PolicyAdjustment schema and generation algorithm
- AdaptivePolicy schema and generation algorithm
- PolicyDimension and AdjustmentDirection enums
- AdaptivePolicyEngine interface
- TelemetryCollector interface
- RuleEvaluator interface
- Preset adaptive policies (default, aggressive)

## What's NOT Frozen

- Telemetry collection mechanisms
- Rule threshold values (tunable)
- Policy adjustment application logic
- Integration with external evaluation systems
